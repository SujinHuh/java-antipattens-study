# 헤렌 커피챗 니키님 면접 힌트 (녹음본 원본 내용)

이 문서는 헤렌 커피챗 당시 니키님이 직접 말씀하신 1차 기술면접 레거시 코드 리뷰 및 이력서 질문에 대한 **100% 원본 힌트 내용**만을 정리한 것입니다.

---

## 1. 면접 진행 방식 및 환경
1. **면접 방식**: 별도의 코딩 테스트나 사전 과제는 없으며, 예전에 사용했던 면접용 레거시 코드 템플릿을 현장에서 보여준다.
2. **평가 방식**: 면접자는 해당 코드를 보고 코드 리뷰하듯이 문제점을 분석하고 지적하면 된다.
3. **코드 분량**: 처음 보면 당황스러울 수 있지만, 로직 자체는 `Controller → Service → Repository → Util`로 이어지는 단순한 흐름이며 코드의 양도 그리 많지 않다.
4. **시간 제한**: 코드를 검토하는 시간은 1시간 이상 넉넉히 주는 것은 아니며, **약 40분 정도** 제공된다.
5. **질문 권장**: 코드 분석 중 도메인 의도나 흐름 상 이해가 안 되는 부분은 혼자 오래 고민하지 말고, **면접관에게 편하게 질문**하면서 파악하는 것을 권장한다.

---

## 2. 합격/불합격의 기준
1. **코드 템플릿 통과 기준**: 레거시 코드 템플릿 리뷰에서 최소한 **중간 이상**은 해내야 다음 단계로 진행할 수 있다.
2. 만약 코드 템플릿 리뷰 결과가 기준 미달일 경우, 이력서나 면접 분위기 등 다른 부분의 느낌이 아무리 좋아도 다음 단계로 넘어가지 못하고 탈락할 수 있다.

---

## 3. 대표적인 코드 결함 예시
니키님이 직접 예시로 든 레거시 코드의 결함 포인트는 다음과 같다.
- **Controller**: `@Valid` 같은 요청값 검증 어노테이션 누락
- **Service**: 비즈니스 로직에 당연히 있어야 할 `@Transactional` 어노테이션 누락

---

## 4. 이력서 기반 기술 질문 병행
기술 면접은 레거시 코드 리뷰뿐만 아니라, 제출한 이력서 기반의 꼬리 질문도 까다롭게 들어올 수 있다.
- **예시**: 이력서에 "GraphQL 도입 경험"이 있다면, *"GraphQL을 왜 도입했는가?"*, *"기존 REST API로는 왜 부족했는가?"* 와 같은 심화 질문이 이어질 수 있다.

---

## 5. 니키님이 권장한 연습 방법
- **AI 활용 연습**: AI에게 문제를 만들어 달라고 해서 연습하는 것을 추천한다.
- 수진이가 *"랜덤한 레거시 코드를 보여달라고 AI에게 요청해서 연습하면 되느냐"*고 질문했을 때, 니키님이 **"그 방식도 괜찮다"**고 직접 동의하고 확인해 주었다.
