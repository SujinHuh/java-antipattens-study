# 18. AI 역할 분담 가이드

이 문서는 Java/Spring Boot 면접 대비 학습 과정에서 어떤 작업을 어떤 AI 역할 또는 에이전트 관점으로 맡길지 판단하기 위한 가이드다.

특정 모델 이름에 고정하지 않고, 역할 중심으로 구분한다.

## 1. 코드 작업 에이전트

코드 기반 문제 생성, 코드 리뷰, 리팩토링, 안티패턴 지침 초안 작성에 사용한다.

주로 맡길 작업:

- Spring MVC 코드 스니펫 생성
- 레거시 코드 문제 생성
- Controller / Service / Repository 문제 생성
- Transaction / AOP 안티패턴 생성
- 사용자가 답한 코드 리뷰 검토
- 리팩토링 방향 제안
- 자바/Spring 코딩 지침 초안 생성

같이 볼 문서:

- 문제 생성: `1. 문제 생성 에이전트.md`, `13. 실제 루프 실행 프롬프트.md`, `17. 루프 상태 기록.md`
- 코드 리뷰/리팩토링: `16. 통합 코드리뷰-리팩토링 코치.md`, 필요 시 `2. 답변 채점 에이전트.md`
- 안티패턴 지침 초안: `7. 안티패턴 지침화 에이전트.md`

트리거 문구:

🚨

## 2. 답변 정리 / 면접 표현 에이전트

사용자의 답변을 면접장에서 말하기 좋게 다듬고, 모호한 표현을 정확한 표현으로 바꾸는 데 사용한다.

주로 맡길 작업:

- 내 답변을 면접용으로 다듬기
- 모호한 표현 잡기
- 30초 / 1분 답변 만들기
- 설명 쉽게 풀기
- 꼬리질문 만들기
- TIL 정리
- 프롬프트 보강

같이 볼 문서:

- 답변 채점/표현 교정: `2. 답변 채점 에이전트.md`
- 개념 설명: `3. 설명 에이전트.md`
- 면접 답변 문장화: `11. 코드 리뷰 면접 답변 템플릿.md`
- TIL 기록: `6. TIL-로그 에이전트.md`
- 다음 문제 보강: `5. 프롬프트 개선 에이전트.md`, `17. 루프 상태 기록.md`

트리거 문구:

## 3. 교차검증 / 다른 관점 에이전트

AI가 만든 문제, 답변, 설명, 피드백이 기술적으로 맞는지 다른 관점에서 검증할 때 사용한다.

주로 맡길 작업:

- 코드 작업 에이전트 답변 검증

- 기술적으로 틀린 부분 찾기
- 실무 관점 추가
- 면접에서 위험한 표현 찾기
- 다른 대안 / 트레이드오프 제시

같이 볼 문서:

- 회사 공고 기반 보강 관점: `19. 헤렌 공고 기반 docs 보강 예정.md`

## 사용 원칙

- 코드 생성과 코드 냄새 탐지는 코드 작업 에이전트 관점으로 먼저 본다.
- 면접장에서 말할 문장과 표현은 답변 정리 / 면접 표현 에이전트 관점으로 다시 본다.
- 기술 판단이 애매하거나 단정적인 설명은 교차검증 / 다른 관점 에이전트로 확인한다.
- 최종 판단은 AI가 아니라 사용자가 한다.
