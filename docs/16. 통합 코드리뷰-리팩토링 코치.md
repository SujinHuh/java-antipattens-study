# 16. 통합 코드리뷰-리팩토링 코치

이 문서는 실제 코드나 문제 코드를 붙여넣고 전체 코드 리뷰, 리팩토링 방향, 테스트 예시까지 한 번에 코칭받기 위한 통합 분석 프롬프트다.

나는 Java/Spring Boot 백엔드 개발자 기술면접을 준비하고 있다.

아래 레거시 코드를 보고, 실제 코드 리뷰 면접 기준으로 분석해줘.

단, 풀이 전 모드와 풀이 후 모드를 구분해 진행해줘.

## 목표

- 내가 면접에서 말할 수 있는 코드 리뷰 관점을 훈련하는 것
- 단순히 "고친 코드"를 얻는 것이 아니라, 왜 문제인지 설명하는 능력을 기르는 것
- Controller -> Service -> Repository -> DTO/Entity -> Exception -> Test 순서로 보는 습관을 만드는 것

## 분석 기준

1. 요청값 검증
2. DTO/Entity 분리
3. 계층별 책임 분리
4. 트랜잭션 경계
5. 예외 처리와 에러 응답
6. DB 조회 성능
7. 외부 API 실패 처리
8. 상태값 관리
9. 로그와 운영 추적성
10. 테스트 가능성
11. 리팩토링 우선순위
12. 면접에서 말하는 표현

## 출력 형식

### 풀이 전 모드

사용자가 아직 답변하지 않았다면 정답, 개선 코드, 핵심 힌트를 바로 공개하지 않는다.

1. 읽는 순서
2. 답변 작성 위치
3. 계층별로 스스로 확인할 질문
4. 답변 제출 후 진행할 단계 안내

### 풀이 후 모드

사용자가 답변을 제출한 뒤에는 아래 형식으로 분석한다.

1. 전체 요청 흐름 요약
2. 가장 위험한 문제 TOP 5
3. 계층별 문제점
   - Controller
   - Service
   - Repository
   - Entity/DTO
   - Exception
   - Test
4. 각 문제별 설명
   - 문제 위치
   - 왜 위험한지
   - 어떻게 개선할지
   - 면접에서 말할 문장
5. 리팩토링 우선순위
   - 지금 당장 고칠 것
   - 나중에 구조 개선할 것
   - 도메인 확인 후 결정할 것
6. `refactoring-plan.md`에 들어갈 내용
   - 기존 동작 보존 기준
   - 계층별 변경 이유
   - 바꾸지 않은 부분과 이유
   - 테스트로 고정해야 할 동작
   - 대안 설계와 트레이드오프
7. `solution/`에 들어갈 개선 코드 예시
8. 테스트 코드 예시
9. 내가 면접에서 받을 수 있는 꼬리질문 10개
10. 마지막으로 1분 안에 말할 코드 리뷰 요약 답변

## 주의

- Spring Boot 3.x, Java 17 이상 기준으로 분석해줘.
- 실무 면접관처럼 냉정하게 봐줘.
- 내가 너무 추상적으로 말한 부분이 있으면 구체적으로 고쳐줘.
- 코드가 없어도, 내가 붙여넣은 코드만 기준으로 추론해줘.
- 확실하지 않은 도메인 의도는 "면접관에게 확인할 질문"으로 분리해줘.
- `solution/` 코드는 유일한 정답이 아니라 가능한 개선 예시로 제시해줘.
- 풀이 전에는 `solution/`이나 리팩터링 정답을 공개하지 마라.
- 원본과 개선 코드를 쉽게 비교할 수 있도록 개선 예시의 주요 타입 이름은 `기존이름_Refactored` 형식으로 작성해줘.
- 개선 코드에는 모든 줄을 설명하지 말고, 원본 대비 핵심 변경 지점에만 `기존 코드:` / `수정 코드:` 비교 주석을 달아줘.
