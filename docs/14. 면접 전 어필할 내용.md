# 14. 면접 전 어필할 내용

## 기본 문장

이번 면접 준비를 하면서 단순히 예상 질문을 뽑아보는 방식이 아니라,
문제를 풀고 답변을 제출한 뒤, 틀린 점과 모호한 표현을 피드백받고,
그 피드백을 다시 문제 생성 프롬프트에 반영하는 루프를 만들어 준비했습니다.

아직 답변이 부족한 부분은 제 한계이지만,
이런 식으로 학습 과정을 개선하는 방식은 앞으로도 계속 활용할 수 있을 것 같습니다.

특히 레거시 코드나 기존 프로젝트에 투입될 때도,
코드와 아키텍처 문서를 기반으로 정책과 예외 흐름을 정리하고,
피드백 루프를 통해 문서를 보강하면서 도메인을 빠르게 파악하는 방식으로 적용해볼 수 있다고 생각합니다.

## 30초 답변

이번 면접 준비에서는 예상 질문을 단순히 많이 보는 방식보다, 문제를 풀고 답변을 피드백받은 뒤 그 약점을 다음 문제에 다시 반영하는 학습 루프를 만들어 연습했습니다. 특히 제가 자주 모호하게 말하는 표현과 Spring 트랜잭션, AOP, 예외 처리 같은 약점을 반복해서 점검했습니다.

## 1분 답변

이번 면접 준비를 하면서 AI를 단순히 예상 질문 생성 도구로만 쓰지는 않았습니다. 레거시 코드 리뷰 문제를 만들고, 제가 직접 답변한 뒤, 틀린 부분과 모호한 표현, 놓친 본질을 분리해서 피드백받는 방식으로 학습 루프를 만들었습니다. 그 피드백을 다시 다음 문제 생성 조건에 반영해서 같은 약점이 반복 출제되도록 했습니다. 이 과정을 통해 Spring MVC, AOP, 트랜잭션 경계, 예외 처리, Java 안티패턴처럼 제가 설명을 흐리게 하기 쉬운 부분을 더 정확한 용어로 말하는 연습을 했습니다.

## 안전한 표현

- "AI가 검증했습니다"보다는 "AI 피드백을 참고하되, 최종 판단은 공식 문서와 코드 기준으로 확인했습니다"라고 말한다.
- "AI로 공부했습니다"보다는 "문제 풀이, 피드백, 재출제 구조로 학습 과정을 반복 개선했습니다"라고 말한다.
- "자동화했습니다"보다는 "제가 놓치는 표현과 개념을 반복적으로 드러내는 보조 도구로 사용했습니다"라고 말한다.

## 피해야 할 표현

- "AI가 맞다고 했습니다"
- "AI가 검증해서 괜찮습니다"
- "AI로 다 자동화했습니다"
- "질문만 뽑아서 외웠습니다"

## 프로젝트 적용 예시

레거시 프로젝트에 투입된다면 코드와 아키텍처 문서를 먼저 훑고, Controller-Service-Repository 흐름, 예외 처리 정책, 트랜잭션 경계, 외부 연동 지점을 정리하겠습니다. 이후 제가 이해한 내용을 문서화하고, 불명확한 정책은 질문 목록으로 분리해서 검증하면서 온보딩 속도를 높일 수 있다고 생각합니다.
